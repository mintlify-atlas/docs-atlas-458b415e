This guide will help you create your first Markdoc document and understand the core workflow: **parse → transform → render**.

## The Markdoc Pipeline

Markdoc processes content in three steps:

<Steps>
  <Step title="Parse">
    Convert your Markdown string into an Abstract Syntax Tree (AST)
  </Step>
  
  <Step title="Transform">
    Transform the AST into a renderable tree using your configuration
  </Step>
  
  <Step title="Render">
    Render the tree to React components or HTML strings
  </Step>
</Steps>

## Basic Example

Here's a complete working example showing the full pipeline:

```javascript
import Markdoc from '@markdoc/markdoc';

const doc = `
# Markdoc README

{% image src="/logo.svg" /%}
`;

// Step 1: Parse - Convert Markdown to AST
const ast = Markdoc.parse(doc);

// Step 2: Transform - Convert AST to renderable tree
const content = Markdoc.transform(ast);

// Step 3: Render - Output depends on your renderer choice
```

## Rendering Options

### React Renderer

Render your content as React components:

```jsx
import Markdoc from '@markdoc/markdoc';
import React from 'react';

const doc = `
# Welcome to Markdoc

This is a **bold** statement with _emphasis_.
`;

const ast = Markdoc.parse(doc);
const content = Markdoc.transform(ast);

// Render to React
const reactNode = Markdoc.renderers.react(content, React);

// Use in your React component
function MyComponent() {
  return <div>{reactNode}</div>;
}
```

### HTML Renderer

Render your content as an HTML string:

```javascript
import Markdoc from '@markdoc/markdoc';

const doc = `
# Welcome to Markdoc

This is a **bold** statement with _emphasis_.
`;

const ast = Markdoc.parse(doc);
const content = Markdoc.transform(ast);

// Render to HTML string
const html = Markdoc.renderers.html(content);

console.log(html);
// Output: <h1>Welcome to Markdoc</h1><p>This is <strong>bold</strong> statement with <em>emphasis</em>.</p>
```

## Creating Custom Tags

One of Markdoc's most powerful features is the ability to define custom tags. Here's how to create a simple callout component:

<Steps>
  <Step title="Define the Tag Schema">
    Create a schema that defines your tag's behavior
  </Step>
  
  <Step title="Add Tag to Config">
    Pass your tag definitions in the config object
  </Step>
  
  <Step title="Use in Markdown">
    Use your custom tag in your Markdoc content
  </Step>
</Steps>

### Complete Custom Tag Example

```javascript
import Markdoc from '@markdoc/markdoc';
import Tag from '@markdoc/markdoc';

// Define a custom callout tag
const callout = {
  render: 'Callout',
  attributes: {
    type: {
      type: String,
      default: 'info',
      matches: ['info', 'warning', 'error']
    },
    title: {
      type: String
    }
  }
};

// Create your config
const config = {
  tags: {
    callout: callout
  }
};

// Use the tag in your content
const doc = `
# Documentation

{% callout type="warning" title="Important" %}
This is a warning message that will stand out!
{% /callout %}
`;

// Process with config
const ast = Markdoc.parse(doc);
const content = Markdoc.transform(ast, config);
```

## Advanced: Custom Tag with Transform

For more control, you can define a `transform` function that customizes how your tag is rendered:

```javascript
import Markdoc from '@markdoc/markdoc';
import Tag from '@markdoc/markdoc';

const heading = {
  children: ['inline'],
  attributes: {
    level: { type: Number, render: false, required: true }
  },
  transform(node, config) {
    return new Tag(
      `h${node.attributes['level']}`,
      node.transformAttributes(config),
      node.transformChildren(config)
    );
  }
};

const config = {
  nodes: {
    heading: heading
  }
};
```

## Working with Conditional Logic

Markdoc includes built-in tags for conditional rendering:

```javascript
const doc = `
{% if $showWarning %}
  # Warning!
  This content is only visible when showWarning is true.
{% else %}
  # All Clear
  Everything is fine.
{% /if %}
`;

const ast = Markdoc.parse(doc);

// Transform with variables
const content = Markdoc.transform(ast, {
  variables: {
    showWarning: true
  }
});
```

## Using the Markdoc Class

For projects with consistent configuration, you can instantiate the Markdoc class:

```javascript
import Markdoc from '@markdoc/markdoc';

// Create a Markdoc instance with your config
const markdoc = new Markdoc({
  tags: {
    callout: {
      render: 'Callout',
      attributes: {
        type: { type: String }
      }
    }
  }
});

// Parse, transform, and validate with your config
const ast = markdoc.parse('# Hello');
const content = markdoc.transform(ast);
const errors = markdoc.validate(ast);
```

## Complete React Example

Here's a full example integrating Markdoc with React and custom components:

```jsx
import Markdoc from '@markdoc/markdoc';
import React from 'react';

// Define custom React component
function Callout({ type, title, children }) {
  return (
    <div className={`callout callout-${type}`}>
      {title && <h4>{title}</h4>}
      {children}
    </div>
  );
}

// Configure Markdoc
const config = {
  tags: {
    callout: {
      render: 'Callout',
      attributes: {
        type: { type: String },
        title: { type: String }
      }
    }
  }
};

// Your Markdoc content
const doc = `
# Getting Started

{% callout type="info" title="Welcome" %}
Thanks for using Markdoc!
{% /callout %}
`;

// Render function
function MarkdocRenderer() {
  const ast = Markdoc.parse(doc);
  const content = Markdoc.transform(ast, config);
  
  return Markdoc.renderers.react(content, React, {
    components: {
      Callout: Callout
    }
  });
}

export default MarkdocRenderer;
```
